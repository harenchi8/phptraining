
# 第13週 実践課題

**目的:**
これまでの学習の集大成として、自分だけのオリジナルWebアプリケーションの企画・設計を行い、開発の土台を構築します。この課題は、コーディングよりも「何をどのように作るか」を考える思考プロセスに重点を置きます。

---

## 課題1: アプリケーションの企画と要件定義 (11/15)

**手順:**

1.  **テーマの決定**:
    *   `lecture.md`の例を参考に、自分が作ってみたいWebアプリケーションのテーマを一つ選びます。自分が普段使っていて「こうだったら良いのに」と思うものや、自分の興味がある分野に関連するものがおすすめです。
    *   例: 「読書記録アプリ」「映画レビューサイト」「学習記録アプリ」「シンプルな家計簿」など。

2.  **企画書の作成**:
    *   `planning.md` のようなマークダウンファイルを作成し、以下の項目を記述してください。
        *   **アプリ名**: アプリケーションの名前（仮でOK）
        *   **概要**: どんなアプリか、1〜2文で簡潔に説明してください。
        *   **目的・ターゲットユーザー**: 「誰が」「何を解決するために」このアプリを使うのかを記述してください。
        *   **コア機能**: このアプリの最も中心的で、最低限実装したい機能を3〜5個リストアップしてください。

3.  **機能要件の洗い出し**:
    *   企画書を元に、必要な機能を「ユーザー機能」と「管理者機能」（もしあれば）に分けて、箇条書きで詳細に洗い出してください。
    *   **例（読書記録アプリ）**:
        *   **ユーザー機能**
            *   ユーザー登録、ログイン、ログアウトができる。
            *   読んだ本を検索して登録できる。（外部の書籍APIを使うか、手動登録にするかなども考える）
            *   本の読書状況（未読、読書中、読了）を管理できる。
            *   本ごとに感想や評価を記録できる。
            *   自分が登録した本の一覧を閲覧できる。
            *   ...など

## 課題2: データベース設計と画面設計 (11/15)

**手順:**

1.  **データベース設計 (ER図)**:
    *   洗い出した機能要件を満たすために、どのようなデータが必要かを考え、テーブル構造を設計します。
    *   手書き、または [draw.io](https://app.diagrams.net/) (現diagrams.net) や [Cacoo](https://cacoo.com/ja/) などのツールを使って、ER図を作成してください。
    *   各テーブルのカラム名、データ型、制約（主キー、外部キーなど）を定義します。
    *   テーブル間のリレーション（1対多など）を線で結びます。
    *   作成したER図は、画像としてエクスポートするか、スクリーンショットを撮ってリポジトリに保存できるようにしておきましょう。

2.  **画面設計 (ワイヤーフレーム)**:
    *   主要な画面（最低でも3〜4画面）のワイヤーフレームを作成します。
    *   手書きのスケッチで十分ですが、Figmaなどのデザインツールを使っても構いません。
    *   「どこに」「何の要素が」「どのように」配置されるかが分かればOKです。完璧なデザインである必要はありません。
    *   **対象画面の例**: 一覧画面、詳細画面、新規登録画面、編集画面など。

## 課題3: 開発の土台構築 (11/16)

**手順:**

1.  **Laravelプロジェクトの作成**:
    *   `composer create-project laravel/laravel final-app` のように、最終課題用の新しいLaravelプロジェクトを作成します。

2.  **初期設定と認証機能の導入**:
    *   `.env` ファイルを編集し、データベース接続設定を行います。（`final_app_db` のような新しいDB名にしましょう）
    *   phpMyAdminで対応するデータベースを作成します。
    *   Laravel Breezeをインストールし、認証機能をセットアップします (`composer require`, `php artisan breeze:install`, `npm install`, `npm run build`)。

3.  **マイグレーションファイルの作成**:
    *   作成したER図に基づいて、必要なモデルとマイグレーションファイルを `php artisan make:model ModelName -m` コマンドで一通り作成します。
    *   各マイグレーションファイルの `up` メソッドを編集し、設計通りにカラムを定義します。

4.  **シーダーの作成**:
    *   開発をスムーズに進めるため、初期データを投入するシーダーを作成します。
    *   `php artisan make:seeder UserSeeder`, `php artisan make:seeder PostSeeder` のように、主要なテーブル用のシーダーを作成します。
    *   `database/seeders/DatabaseSeeder.php` の `run` メソッドから、作成した各シーダーを呼び出すように設定します。

5.  **マイグレーションとシーディングの実行**:
    *   以下のコマンドを実行し、テーブル作成と初期データの投入を一括で行います。
        ```bash
        php artisan migrate:fresh --seed
        ```
    *   phpMyAdminで、設計通りのテーブルが作成され、ダミーデータが投入されていることを確認します。

**今週の成果物:**
- `planning.md` (企画書)
- ER図の画像ファイル
- ワイヤーフレームの画像ファイル（またはスケッチ）
- マイグレーションとシーダーが準備されたLaravelプロジェクト

これらの設計資料とプロジェクトの土台をGitHubのリポジトリにプッシュして、今週の課題は完了です。
