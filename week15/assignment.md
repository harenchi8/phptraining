
# 第15週 実践課題

**目的:**
開発したアプリケーションの品質を向上させ、ポートフォリオとして公開できる状態に仕上げます。UI/UXの改善、本番環境へのデプロイ、そしてプロジェクトの顔となるREADMEの作成を行います。

---

## 課題1: UI改善と本番化 (11/29)

**手順:**

1.  **UI/UXの改善**:
    *   **レスポンシブ対応**: Tailwind CSSのブレークポイント (`sm:`, `md:`, `lg:`) を活用して、スマートフォン表示でもレイアウトが崩れず、快適に操作できるようにスタイルを調整します。特に、テーブルや横長の要素が画面からはみ出さないように注意しましょう。
    *   **Bladeコンポーネント化**: アプリケーション内で共通して使っているUI部品（ボタン、カード、フォーム要素など）を、`php artisan make:component` コマンドでコンポーネントとして切り出しましょう。ビューのコードが `<x-button-primary>...</x-button-primary>` のようにシンプルになり、見通しが良くなることを体験してください。
    *   **フィードバックメッセージ**: 投稿・更新・削除などの操作が完了した際に、「投稿が完了しました。」のようなフラッシュメッセージがすべての操作で表示されるようになっているか確認し、実装漏れがあれば追加します。
    *   **ナビゲーション**: 全てのページに共通のヘッダー（ナビゲーションバー）を設置し、どのページからでも主要な機能（トップページ、ログイン/ログアウト、新規投稿など）にアクセスできるようにします。Breezeが提供するレイアウトファイル (`resources/views/layouts/app.blade.php`) を継承 (`@extends`) するのが簡単です。

2.  **本番環境へのデプロイ**:
    *   第12週の課題を参考に、作成したアプリケーションをRender.com（またはHerokuなど他のPaaS）にデプロイします。
    *   **デプロイプロセス:**
        1.  GitHubに最新のコードをプッシュします。
        2.  RenderのダッシュボードでWeb Serviceを作成し、GitHubリポジトリを連携します。
        3.  ビルドコマンド (`composer install ...`) とスタートコマンド (`php artisan serve ...`) を設定します。
        4.  Render上でデータベースを作成します。
        5.  `.env` の内容を、本番用の値（`APP_ENV=production`, `APP_DEBUG=false`, 本番DBの接続情報など）に書き換えて、RenderのEnvironment Variablesに設定します。
        6.  デプロイを実行します。
        7.  デプロイが完了したら、RenderのShell機能を使って、本番環境でマイグレーションを実行します。
            ```bash
            php artisan migrate --seed # 必要であれば --seed も
            ```
        8.  ファイルアップロード機能がある場合は、`storage:link` も実行する必要があるか、PaaSのドキュメントを確認します。（Renderではビルドスクリプトに `php artisan storage:link` を含めるのが一般的です）
    *   発行されたURLにアクセスし、アプリケーションが本番環境で正しく動作することを確認します。特に、画像の表示やDBとの連携でエラーが出やすいので注意深く確認してください。

## 課題2: 成果発表用まとめ (11/30)

**目的:**
作成したアプリケーションを、あなたのスキルセットを証明するポートフォリオとして完成させます。

**手順:**

1.  **スクリーンショットの撮影**:
    *   アプリケーションの主要な画面（トップページ、一覧、詳細、フォームなど）のスクリーンショットを撮影します。ブラウザのデベロッパーツールを使って、PC表示とスマートフォン表示の両方を撮影すると、レスポンシブ対応をアピールできます。

2.  **README.mdの清書**:
    *   `lecture.md` の「READMEに含めるべき項目」を参考に、プロジェクトのルートにある `README.md` を丁寧に記述します。
    *   **文章だけでなく、箇条書きやコードブロック、画像を効果的に使って、見やすく分かりやすいドキュメントを作成しましょう。**
    *   特に「使用技術」と「機能一覧」は、あなたのスキルを具体的に示す重要な項目です。
    *   ローカルでの実行方法を記述することで、他の開発者に対する配慮も示すことができます。

3.  **GitHubリポジトリの整理**:
    *   リポジトリの「About」セクションに、アプリケーションの簡単な説明と、デプロイ先のURLを記載します。
    *   関連するトピック（例: `php`, `laravel`, `mysql`, `portfolio`）をタグとして設定しておくと、見つけてもらいやすくなります。

4.  **最終確認と提出**:
    *   デプロイしたURLが正常に動作すること。
    *   GitHubリポジトリが整理され、READMEが完成していること。
    *   これらを確認して、最終課題の完了となります。
