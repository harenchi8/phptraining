### レスポンシブデザイン

現代のWebサイトは、PC、タブレット、スマートフォンなど、様々な画面サイズのデバイスからアクセスされます。レスポンシブデザインは、CSSを使って、デバイスの画面幅に応じてレイアウトやスタイルを最適化する手法です。

**CSSメディアクエリ**

特定の条件下でのみCSSを適用させる仕組みです。画面幅を条件に使うことが最も一般的です。

```css
/* デフォルトはモバイル用のスタイルを記述 (モバイルファースト) */
.container {
    width: 100%;
    padding: 15px;
}

/* 画面幅が768px以上の場合 (タブレットなど) */
@media (min-width: 768px) {
    .container {
        width: 750px;
        margin: 0 auto;
    }
}

/* 画面幅が992px以上の場合 (PCなど) */
@media (min-width: 992px) {
    .container {
        width: 970px;
    }
}
```

**CSSフレームワークの活用**

Breezeをインストールした際に導入された **Tailwind CSS** は、レスポンシブデザインを非常に効率的に実装できるユーティリティファーストのCSSフレームワークです。

- `md:text-lg` のように、クラス名の前に `md:` (medium) や `lg:` (large) といったプレフィックス（ブレークポイント）を付けることで、特定の画面幅以上でのみスタイルを適用できます。

```html
<!-- モバイルでは文字が小さく、md(768px)以上では大きくなる -->
<h1 class="text-base md:text-xl">タイトル</h1>

<!-- モバイルでは縦並び、lg(1024px)以上では横並びになるフレックスボックス -->
<div class="flex flex-col lg:flex-row">
    <div>項目A</div>
    <div>項目B</div>
</div>
```

### Bladeコンポーネントによる共通化

アプリケーション内で繰り返し使われるUI部品（ボタン、フォームの入力欄、カードなど）は、**Bladeコンポーネント**として切り出すと、再利用性が高まり、コードの見通しが良くなります。

**コンポーネントの作成:**
```bash
php artisan make:component ButtonPrimary
```
- `resources/views/components/button-primary.blade.php` が作成されます。

**コンポーネントの定義 (`button-primary.blade.php`):**
```blade
<button {{ $attributes->merge(['type' => 'submit', 'class' => 'px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600']) }}>
    {{ $slot }} {{-- ここにボタンのテキストが入る --}}
</button>
```
- `$attributes->merge([...])` : コンポーネント呼び出し側で指定された属性をマージします。
- `$slot`: コンポーネントの開始タグと終了タグで囲まれた中身がここに展開されます。

**コンポーネントの利用:**
```blade
<x-button-primary>
    更新する
</x-button-primary>

<x-button-primary type="button" class="bg-red-500">
    削除する
</x-button-primary>
```

### READMEの作成

READMEは、そのプロジェクトの「顔」です。他の人がリポジトリを見たときに、どんなプロジェクトで、どうやって動かすのかがすぐに分かるように、丁寧に記述することが重要です。これはポートフォリオとして他人に見せる際に非常に役立ちます。

**READMEに含めるべき項目:**
- **アプリケーション名と概要**: 何のアプリかを簡潔に説明。
- **スクリーンショット**: 主要な画面のスクリーンショットを数枚貼り、視覚的にアプリのイメージを伝えます。
- **URL**: デプロイしたアプリケーションのURL。
- **主な機能**: 実装した機能の一覧を箇条書きでアピールします。
- **使用技術**: 使用した言語、フレームワーク、ライブラリなどを記載します。(例: PHP 8.1, Laravel 9.x, MySQL, Tailwind CSS, JavaScript)
- **ER図**: データベース設計が分かるER図を貼り付けます。
- **インストール・実行方法**: ローカル環境でこのプロジェクトを動かすための手順を記述します。
    1. `git clone ...`
    2. `composer install`
    3. `.env.example` をコピーして `.env` を作成
    4. `php artisan key:generate`
    5. `php artisan migrate --seed`
    6. `npm install && npm run build`
    7. `php artisan serve`

### ポートフォリオとしての見せ方

作成したアプリケーションは、あなたのスキルを証明する貴重なポートフォリオになります。

- **コードの品質**: 他人が読んでも分かりやすいように、変数名やメソッド名を適切に付け、インデントを揃え、不要なコメントは削除するなど、コードを綺麗に保ちましょう。
- **コミット履歴**: `feat: ...`, `fix: ...` のような規約に沿った分かりやすいコミットメッセージは、開発プロセスを丁寧に管理できることのアピールになります。
- **GitHubのプロフィール**: GitHubのプロフィールページも綺麗に整えておくと、採用担当者などへの印象が良くなります。
